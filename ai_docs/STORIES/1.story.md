# WAVELET_RESEARCH_STORY_01.md

# Story

Implement the data ingestion layer for the Wavelet Research Platform.

---

# Goal

Create a deterministic, production-quality loader capable of importing raw MT5 tick exports.

This module becomes the single source of truth for all future research.

No trading logic should be implemented in this story.

---

# Input

Support both formats.

## Format A

Normalized CSV

```text
time,bid,ask
```

## Format B

Raw MT5 Tick Export

```text
<DATE>    <TIME>    <BID>    <ASK>    <LAST>    <VOLUME>    <FLAGS>
```

Separator:

```text
TAB
```

The loader must detect the format automatically.

---

# Output

Normalized dataframe.

Columns:

```text
time
bid
ask
mid
spread
flags
```

Derived fields:

```text
mid = (bid + ask) / 2
spread = ask - bid
```

---

# Parsing

Requirements:

- Parse timestamps with millisecond precision.
- Combine `<DATE>` and `<TIME>`.
- Preserve original tick order.
- Support files containing millions of ticks.

---

# Bid / Ask Handling

MT5 exports frequently contain updates where only Bid or only Ask changes.

Implementation requirements:

- Forward-fill Bid independently.
- Forward-fill Ask independently.
- Drop rows only after forward filling if Bid or Ask is still missing.

Never discard partial updates before forward filling.

---

# Validation

Reject data when:

- ask < bid
- invalid timestamp
- non-numeric Bid/Ask

Duplicate timestamps are allowed if market values differ.

---

# Ignore

Ignore for now:

```text
LAST
VOLUME
```

Keep FLAGS.

---

# Performance

Implementation must:

- Use vectorized pandas operations.
- Avoid Python loops.
- Scale to 10M+ ticks.
- Minimize unnecessary dataframe copies.

---

# Public API

ticks = load_ticks(path)

ticks.time
ticks.bid
ticks.ask
ticks.mid
ticks.spread
ticks.flags
```
