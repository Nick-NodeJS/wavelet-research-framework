# WAVELET_RESEARCH_STORY_16.md

# Story

Implement the first production Python Wavelet Service for MT5 integration.

---

# Goal

Build a local Python service exposing the existing Wavelet Engine through a stable API.

This service becomes the single computation backend for:

- MT5 Indicator
- MT5 Expert Advisor
- Future Research Tools

No wavelet calculations may be implemented in MQL5.

---

# Scope

Implement:

- Local service
- Request validation
- Wavelet calculation endpoint
- Streaming-ready architecture
- Configuration loading
- Health endpoint

Do NOT implement:

- MT5 integration
- Trading logic
- Signal generation
- Order execution

---

# Wavelet

Use only:

- db4

No optimization.

No alternative wavelets.

No experimentation.

---

# Input

Normalized ticks.

```json
{
  "ticks": [
    {
      "time": "...",
      "bid": 1.12345,
      "ask": 1.12347,
      "mid": 1.12346
    }
  ]
}
```

---

# Output

```json
{
  "trend": [...],
  "relative_deviation": [...],
  "z_score": [...],
  "energy": [...],
  "noise": [...]
}
```

All arrays must have identical length.

---

# Relative Deviation

Implement

```
relative_deviation =
(mid - trend) / local_volatility
```

The normalization strategy must be configurable.

---

# Service API

Implement:

```
GET  /health

POST /wavelet
```

Health returns:

```json
{
  "status":"ok",
  "wavelet":"db4",
  "version":"..."
}
```

---

# Architecture

Reuse existing modules.

Never duplicate:

- Wavelet Engine
- Validation
- Configuration

The service is only an adapter.

---

# Configuration

Support:

- wavelet=db4
- window
- decomposition_level
- volatility_window

Loaded from configuration.

No hardcoded constants.

---

# Performance

Target:

- <10 ms for a single request with 2000 ticks.

No unnecessary dataframe copies.

No recalculation outside rolling window.

---

# Logging

Structured logging.

Request duration.

Errors.

Wavelet configuration.

---

# Validation

Reject:

- invalid JSON
- missing fields
- insufficient history
- invalid prices

Return meaningful HTTP errors.

---

# Tests

Create:

- endpoint tests
- serialization tests
- deterministic output tests
- performance smoke tests
- configuration tests
- regression tests

---

# Acceptance Criteria

- Service starts locally.
- Existing Wavelet Engine is reused.
- db4 trend is returned correctly.
- Relative Deviation is returned.
- Ready for MT5 integration.
- Fully covered by automated tests.
