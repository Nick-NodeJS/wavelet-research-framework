# WAVELET_RESEARCH_STORY_17.md

# Story

Implement the MT5 Thin Indicator.

---

# Goal

Create an MT5 indicator that acts only as a visualization client.

All mathematical calculations must remain in the Python service.

The indicator is responsible only for:

- collecting market data
- communicating with Python
- drawing returned values

---

# Scope

Implement:

- MT5 Indicator
- HTTP client
- JSON serialization
- Response parsing
- Indicator buffers
- Automatic refresh

Do NOT implement:

- Wavelet calculations
- Signal generation
- Trading logic
- Position management

---

# Architecture

```
MT5 Indicator
        │
        ▼
HTTP Client
        │
        ▼
Python Wavelet Service
        │
        ▼
Wavelet Engine
```

---

# Data Collection

On every recalculation:

Collect latest configurable number of ticks.

Default:

```
2048 ticks
```

Convert every tick into

```json
{
    "time": "...",
    "bid": ...,
    "ask": ...,
    "mid": ...
}
```

---

# HTTP Request

POST

```
/wavelet
```

---

# Expected Response

```json
{
    "trend": [...],
    "relative_deviation": [...],
    "z_score": [...],
    "energy": [...],
    "noise": [...]
}
```

Validate array sizes.

Reject malformed responses.

---

# Visualization

Main chart:

- Trend

Separate window:

- Relative Deviation
- Z-Score
- Energy

Noise may be hidden initially.

---

# Parameters

Indicator inputs:

- Server URL
- Tick Window
- Request Timeout
- Auto Refresh
- Draw Trend
- Draw Relative Deviation
- Draw Z-Score
- Draw Energy

---

# Error Handling

Display readable status:

- Connecting...
- Connected
- Timeout
- Invalid Response
- Service Offline

Indicator must never crash.

---

# Performance

Target latency:

<20 ms

Avoid unnecessary allocations.

Do not block MT5 UI.

---

# Logging

Log:

- request duration
- response size
- HTTP errors
- parsing errors

---

# Tests

Create:

- HTTP client tests
- JSON parser tests
- response validation tests
- indicator buffer tests
- reconnect tests
- timeout tests

---

# Acceptance Criteria

- Indicator connects to local Python service.
- Sends latest ticks.
- Receives trend successfully.
- Draws trend over price.
- Draws Relative Deviation in separate window.
- No wavelet calculations exist in MQL5.
- Stable during continuous market updates.
