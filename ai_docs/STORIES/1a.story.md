# WAVELET_RESEARCH_STORY_01A.md

# Story

Implement a dataset inspection CLI for validating MT5 tick ingestion before any research or trading logic is developed.

---

# Goal

Create a command-line tool that loads tick data using the existing ingestion layer and prints dataset statistics.

The implementation must reuse the existing loader.

No parsing logic may be duplicated.

---

# CLI

```bash
python3 -m wavelet_research.cli_inspect_ticks \
    --input data/EURUSD.csv
```

---

# Requirements

The CLI must:

1. Load data using the existing ingestion API.
2. Perform all existing validations.
3. Print dataset statistics.
4. Exit with code 0 on success.
5. Exit with non-zero code on validation failure.

---

# Output

Print exactly:

```text
raw_rows
normalized_rows

first_timestamp
last_timestamp

bid_missing_before_ffill
ask_missing_before_ffill

bid_missing_after_ffill
ask_missing_after_ffill

invalid_spread_count

duplicate_timestamp_count

min_spread
avg_spread
max_spread
```

---

# Additional Statistics

Also print:

```text
min_bid
max_bid

min_ask
max_ask

time_span

memory_usage_mb
```

---

# Constraints

Do NOT

- duplicate loader logic
- duplicate validation logic
- duplicate normalization logic

Reuse the ingestion package.

---

# Error Handling

Validation errors must:

- display a readable message
- return non-zero exit code

Unexpected exceptions must:

- print stack trace
- return non-zero exit code

---

# Tests

Create tests for:

- valid dataset
- invalid spread
- invalid timestamp
- empty dataset
- duplicated timestamps
- successful CLI execution

---

# Code Quality

- Python 3.12+
- Full type hints
- Small functions
- No duplicated code
- Ruff compatible
- Black compatible
- mypy compatible

---

# Acceptance Criteria

- CLI loads the provided MT5 export.
- Statistics are printed correctly.
- Existing ingestion layer is reused.
- No duplicated parsing or validation logic.
- All tests pass.
